# АС «Планирование рациона питания»

Десктопное приложение для составления индивидуальных планов питания
с автоматическим расчётом КБЖУ и суточной нормы калорий.

Курсовая работа по дисциплине **«Базы данных»**.
МГТУ им. Н.Э. Баумана, кафедра ИУ-5, группа ИУ5-42Б, **Угрюмов Н.А.**, 2026.

---

## Возможности

- Управление справочниками продуктов (с КБЖУ), блюд и категорий.
- Автоматический расчёт КБЖУ блюда и рациона на стороне СУБД.
- Расчёт суточной нормы калорий по формуле **Mifflin-St Jeor** с учётом
  пола, возраста, веса, роста, активности и цели.
- Формирование индивидуальных рационов на интервал дат: приёмы пищи,
  блюда, порции.
- Списки исключаемых продуктов (учитываются при подборе блюд) и
  избранных блюд.
- Аналитика: гистограмма топ-калорийных продуктов и круговая
  диаграмма распределения калорий по пользователям.
- Генерация PDF-отчётов по любой таблице, профилю, рациону и составу блюда.
- Разграничение прав: АРМ Администратора и АРМ Пользователя.

## Архитектура

Трёхзвенная схема:

- **Уровень представления** — PyQt5 (диалог авторизации, главное окно с
  вкладками, диалоги форм, графики, генератор PDF).
- **Уровень доступа к данным** — `db.py`: класс `DB` поверх psycopg2 с
  параметризованными запросами и явным `ROLLBACK` на ошибках.
- **Уровень хранения** — PostgreSQL: 11 таблиц, ограничения
  целостности (`FK`, `CHECK`, `UNIQUE`), два агрегирующих представления
  `dish_nutrition` и `ration_nutrition` для автоматического подсчёта КБЖУ.

## Запуск

### Требования

- Docker 24+ с `docker compose` v2.
- Графическая подсистема:
  - Linux — X11 (по умолчанию);
  - Windows 10/11 — WSL2 c WSLg (поставляется с системой);
  - macOS — XQuartz с разрешённым сетевым подключением.

### Команды

```bash
make build        # собрать образы
make up           # запустить БД и приложение
make logs         # хвост логов
make down         # остановить
make db-reset     # перезалить БД из sql/*.sql
make help         # полный список целей
```

После `make up` открывается окно авторизации. PostgreSQL при первом
запуске автоматически инициализируется из скриптов в `sql/`.
